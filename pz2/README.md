# Практичне виконання ПР2: IoT у Packet Tracer

## 1. Топологія для завдання 2.1

Побудовано IoT-рішення на основі сонячного джерела живлення:

- `PT-Solar Panel` виробляє енергію
- `PT-Power Meter` вимірює передану енергію
- `PT-Battery` накопичує енергію та живить навантаження
- `LED1`-`LED4` підключені як споживачі енергії
- `Switch`, `Server` і `PC` забезпечують мережеву взаємодію, DHCP та доступ до веб-інтерфейсу IoE-сервера

Логіка роботи: сонячна панель передає енергію через лічильник потужності до акумулятора, акумулятор живить світлодіоди, а стан IoT-пристроїв контролюється через сервер `1.0.0.1` з браузера ПК.

## 4. IP-налаштування та IoE-реєстрація

Сервер:

- IP-адреса IoE/DHCP-сервера: `1.0.0.1`
- Обліковий запис IoE-сервера: `admin`
- Пароль IoE-сервера: `admin`
- DHCP-сервіс увімкнений

На IoT-пристроях вибрано DHCP:

- `PT-Solar Panel`: `Config -> GigabitEthernet0 -> DHCP`
- `PT-Power Meter`: `Config -> FastEthernet0 -> DHCP`
- `PT-Battery`: `Config -> FastEthernet0 -> DHCP`

Отримані адреси у виконаному варіанті:

| Пристрій         | IP-адреса | Спосіб отримання | IoE Server                      |
| ---------------- | --------: | ---------------- | ------------------------------- |
| `PC`             | `1.0.0.2` | DHCP             | не реєструється як IoT-пристрій |
| `PT-Solar Panel` | `1.0.0.3` | DHCP             | `1.0.0.1`, `admin/admin`        |
| `PT-Power Meter` | `1.0.0.4` | DHCP             | `1.0.0.1`, `admin/admin`        |
| `PT-Battery`     | `1.0.0.5` | DHCP             | `1.0.0.1`, `admin/admin`        |

Примітка: якщо пристрої підключаються або оновлюють DHCP-оренду в іншому порядку, останній октет може відрізнятися. Головна умова виконання: усі IoT-пристрої мають бути в одній мережі з сервером і мати доступ до `1.0.0.1`.

IoE-реєстрація для кожного з трьох IoT-пристроїв:

1. Відкрити пристрій
2. Перейти до `Config -> Settings`
3. У блоці `IoE Server` вибрати `Remote Server`
4. Ввести `Server Address: 1.0.0.1`
5. Ввести `User Name: admin`
6. Ввести `Password: admin`
7. Натиснути `Connect`

## 5. Перевірка роботи топології

Перевірено такі результати:

- Сонячна панель передає енергію до лічильника потужності
- Лічильник потужності показує отриману/передану енергію
- Акумулятор заряджається від сонячної панелі через лічильник
- `LED1`-`LED4` отримують живлення від акумулятора
- Якщо заряд акумулятора падає, світлодіоди тьмяніють або вимикаються
- З ПК відкрито `Desktop -> Web Browser`
- У браузері введено адресу `1.0.0.1`
- Вхід виконано з логіном `admin` і паролем `admin`
- На сторінці IoE-сервера відображено 3 пристрої:
  - `Connected Solar Panel`
  - `Connected Power Meter`
  - `Connected Battery`

Комутатор, сервер і ПК не відображаються на сторінці IoE-пристроїв, тому що вони є мережевою інфраструктурою або засобами доступу, а не зареєстрованими IoT-пристроями. Світлодіоди в цій схемі працюють як навантаження, підключене до акумулятора, і окремо на IoE-сервері не реєструються.

## 6. Практичні відповіді для завдання 2.2

### 6.1. Переваги датчиків на стоянках у муніципальному гаражі

Датчики на стоянках дають змогу бачити вільні місця в реальному часі, швидше спрямовувати водіїв до доступного місця, зменшувати затори, економити пальне, скорочувати викиди, автоматизувати оплату, контролювати порушення правил паркування та збирати статистику для планування міської інфраструктури.

### 6.2. Дані зі смарт-телевізора та фітнес-пристрою

Смарт-телевізор може передавати дані про переглянуті канали, використані застосунки, час перегляду, пошукові запити, голосові команди, мережеві параметри, ідентифікатори пристрою та вподобання користувача.

Фітнес-пристрій може передавати кількість кроків, пульс, витрачені калорії, тривалість сну, якість сну, GPS-маршрут, дистанцію, темп, тип тренування та інші біометричні показники, які підтримує конкретна модель.

### 6.3. Використання цих даних

Дані можна використати для персоналізації сервісів, рекомендацій контенту, контролю здоров’я, аналізу фізичної активності, планування тренувань, виявлення відхилень у стані організму, оптимізації міських сервісів і покращення якості обслуговування.

Таке використання є добрим, якщо дані збираються прозоро, за згодою користувача, захищаються і використовуються лише для зрозумілої мети. Воно стає поганим, якщо дані збирають без згоди, продають третім сторонам, використовують для стеження, нав’язливої реклами, дискримінації або зберігають без належного захисту.
